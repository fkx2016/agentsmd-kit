# About This Kit

You're holding the **AGENTS.md Starter Kit** — the universal AI-agent context kit.

## If you also use Claude Code

There's a deeper companion kit at **[claudemd.kurkalabs.dev](https://claudemd.kurkalabs.dev)** focused entirely on the Claude Code stack. It includes:

- A four-layer system (`~/.claude/CLAUDE.md`, `<repo>/CLAUDE.md`, `CLAUDE.local.md`, path-scoped `subdir/CLAUDE.md`)
- Templates for common stacks (Cloudflare Worker, Python package, Node + TypeScript, FastAPI, React, monorepo)
- Custom commands (`/standup`, `/postmortem`, `/explain`)
- Agent definitions (`code-reviewer`, `test-writer`)
- A `.claude/` subsystem with rules, skills, and hooks
- Installer scripts for macOS, Linux, and Windows

The two kits are designed to coexist:

- This kit (**agentsmd.kurkalabs.dev**) gives you universal coverage. Every coding agent reads `AGENTS.md`. The one-line `CLAUDE.md` shim makes Claude Code read it too.
- The companion kit (**claudemd.kurkalabs.dev**) goes deep on what Claude Code can do that no other agent can — auto-memory, skills, hooks, the full `.claude/` system.

If you only use Claude Code, you can skip this kit and use the companion alone. If you use anything else (Codex, Cursor, Copilot, Gemini CLI, Aider, etc.) **and** Claude Code, this is the one to start with — and the companion adds the Claude-specific depth on top.
